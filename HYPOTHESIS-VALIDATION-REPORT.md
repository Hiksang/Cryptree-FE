# HyperView 가설 검증 보고서 v3

> PRD v4.0 기반 | 작성일: 2025-02-15
> v2: 핵심 바이럴 루프를 "보상 기대 → 지갑 등록 → 레퍼럴"로 재정의
> v3: 포지셔닝 변경 반영 — "PnL 트래커" → "HyperEVM 트랜잭션 해석 + 온체인 활동 측정 & 보상"

---

## 핵심 전제 수정

### 기존 PRD의 바이럴 루프 (허영심 기반)
```
Identity 카드가 멋있다 → 트위터에 공유한다 → 친구가 본다 → 가입한다
```

### 수정된 바이럴 루프 (경제적 인센티브 기반)
```
"지갑 등록하면 미래에 USDC를 받는다"
  → 지갑을 등록한다
  → "더 많은 보상을 위해 친구를 초대한다"
  → 레퍼럴 링크를 공유한다
  → 친구도 같은 동기로 등록한다
```

**Identity 카드, DeFi DNA는 수단(공유 콘텐츠 포장)이지, 등록의 이유가 아니다.**
등록의 이유는 **"내 온체인 데이터가 수익이 되고, USDC로 돌아온다"**는 약속이다.

이 전제 변경으로 검증해야 할 가설의 구조가 근본적으로 달라진다.

### F1. "미래 USDC 보상의 약속이 지갑 등록을 유도한다"

**PRD 근거:** "기여한 만큼 USDC로 분배" (Section 1.1), 전체 포지셔닝의 핵심

**실제 유저의 의사결정:**
```
유저가 판단하는 것:
  "내 지갑을 등록하면 뭘 얻는가?"
  → 미래에 USDC를 받는다 (보상)
  → HyperEVM 트랜잭션 해석 + 활동 스코어/등급 (유틸리티)
  → 무료 크로스체인 PnL/세금 분석 (부가 유틸리티)

유저가 감수하는 것:
  → 내 온체인 활동이 플랫폼에 노출된다 (프라이버시)
  → 아직 검증 안 된 새 플랫폼을 신뢰해야 한다 (리스크)
```

**왜 치명적인가:**
- 이 약속이 설득력이 없으면 지갑 등록 자체가 안 일어난다
- 등록이 없으면 데이터 없고, 광고 없고, 수익 없고, 분배 없다

**반론 (틀릴 수 있는 이유):**
- **"미래 USDC"는 본질적으로 불확실한 약속이다.** 크립토 유저는 "나중에 준다"는 약속에 수없이 속았다 (FTX, Luna, 수많은 러그풀). 신규 플랫폼의 약속을 왜 믿는가?
- **Phase 0-1 예상 분배: 유저당 $14/분기.** 이 금액을 위해 지갑을 등록할 동기가 되는가? DeFi 파워유저의 기회비용 대비 무시할 수준이다
- **보상이 "시즌 종료 후"에야 지급된다.** 3개월 뒤의 $14를 위해 오늘 행동하는 사람은 적다 (시간 할인 편향)
- **Brave Browser의 선례:** "브라우저 쓰면 BAT 토큰 준다"는 동일한 모델이었지만, BAT 보상은 유저 획득의 핵심 동력이 되지 못했다. 유저는 "더 좋은 브라우저여서" 사용했다

**검증 방법:**
```
실험 1: 랜딩 페이지 A/B 테스트 (Phase 0, Week 1)

  A안: "HyperEVM 온체인 활동 분석" (활동 분석 강조)
  B안: "활동 분석 + USDC 보상" (분석 + 보상)
  C안: "지갑 등록하면 USDC 보상" (보상 강조)

  측정: 지갑 주소 입력 전환율
  → 어느 메시지가 등록을 더 잘 유도하는지 확인
  → 보상 약속의 실제 전환 기여도 분리 측정

실험 2: "보상 없이도 등록하는가?" 테스트

  USDC 보상 언급 없이 "HyperEVM 활동 분석 + 스코어/등급"만으로 등록 유도
  → 등록률이 의미 있으면(> 15%): 활동 분석 자체에 가치 있음. 보상은 가속기
  → 등록률이 낮으면(< 5%): 활동 분석만으로는 부족. 보상 연결 필수

실험 3: 신뢰 장벽 측정

  등록 직전에 이탈하는 유저 대상 이탈 설문:
  "왜 등록하지 않으셨나요?"
  a) 보상을 믿지 않아서
  b) 프라이버시 우려
  c) 필요한 기능이 아니어서
  d) 다른 도구를 이미 사용 중

킬 기준: 보상 강조(C안) 전환율이 활동 분석 강조(A안)보다 낮으면
  → "보상"이 오히려 스캠 인식을 유발하는 것. 활동 분석 우선 전략으로 전환
```

**대안 (틀렸을 때):**
- 미래 보상 대신 **즉시 가치**(HyperEVM tx 해석, 활동 스코어, 등급)를 등록 동기로 전환
- "USDC 분배"를 초기 마케팅에서 빼고, "활동 측정 & 보상"으로 자연스럽게 포지셔닝
- 보상 시점을 "시즌 종료"에서 "월간" 또는 "주간 미니 분배"로 앞당겨 신뢰 구축

### F2. "레퍼럴 보상이 유저를 능동적 전파자로 만든다"

**PRD 근거:** 등급 연동 레퍼럴 보상 (Section 4.1.3), 바이럴 계수 > 1.2 목표

**핵심 메커니즘:**
```
내가 Gold 유저를 초대하면 +800pts
내가 초대한 유저가 누군가를 초대하면 +20pts
→ 더 많은 사람을 초대할수록 내 시즌 보상이 커진다
```

**왜 치명적인가:**
- 유료 마케팅 없이 성장하려면 유저 스스로 퍼뜨려야 한다
- 레퍼럴이 안 되면 CAC(고객획득비용)가 발생하고, 수익 분배할 여력이 줄어든다

**반론 (틀릴 수 있는 이유):**

1. **레퍼럴 피로:** 크립토 유저는 이미 수십 개의 레퍼럴 링크에 노출되어 있다. Binance, Bybit, 각종 DeFi 프로토콜. "또 다른 레퍼럴 링크"에 반응할 여력이 있는가?

2. **네트워크 포화:** 한 유저가 초대할 수 있는 DeFi 유저는 제한적이다. 보통 5-10명. 그중 실제로 가입하는 비율은 10-20%. 즉 유저당 1-2명 초대가 현실적. K-factor = (1.5명 초대) × (50% 가입) = 0.75 < 1.0

3. **보상의 불확실성:** "+800pts"가 실제로 얼마인지 유저가 계산하기 어렵다. 포인트 → USDC 전환율이 시즌 종료까지 모른다. 불확실한 보상으로 사회적 자본(친구에게 추천)을 소비할 동기가 약하다

4. **품질 vs 양의 딜레마:** 등급 연동 보상은 "고래를 데려오라"는 인센티브. 하지만 내 친구 중 Diamond급($2M+ 볼륨) 트레이더가 몇 명인가? 대부분의 레퍼럴은 Bronze(+100pts)가 될 것

5. **스팸 인식:** 레퍼럴 링크를 트위터/디스코드에 올리면 스팸으로 인식된다. DeFi 커뮤니티에서 레퍼럴 링크 공유는 부정적으로 보는 문화가 있다

**검증 방법:**
```
실험 1: 레퍼럴 의향 테스트 (Phase 0, Week 2)

  지갑 등록 완료 후 레퍼럴 코드 제공
  "친구를 초대하면 보상이 커집니다"
  측정:
    - 레퍼럴 링크 복사율: > 30%
    - 실제 공유율 (트위터/디스코드): > 10%
    - 공유당 가입 전환: > 1명

실험 2: 보상 크기 민감도

  그룹 A: "초대하면 포인트 보상" (금액 미명시)
  그룹 B: "초대하면 예상 $5-20 USDC 보상" (금액 명시)
  → 금액 명시가 전환에 도움이 되는가, 오히려 "적다"는 인식을 주는가

실험 3: 채널별 효과

  유저가 어디에 레퍼럴 링크를 공유하는지 추적:
  - 트위터 (공개): 바이럴 계수 기여도
  - 디스코드/텔레그램 (DeFi 그룹): 전환율
  - DM (1:1): 전환율
  → 가장 효과적인 채널에 UX 최적화

킬 기준:
  - 레퍼럴 링크 복사율 < 10%: 레퍼럴 인센티브 자체가 약함
  - 공유당 가입 < 0.3명: 공유해도 전환 안 됨. 콘텐츠/메시지 문제
  - 전체 K-factor < 0.3: 레퍼럴 기반 성장 포기
```

**대안 (틀렸을 때):**
- 레퍼럴 대신 **커뮤니티/파트너십 성장** (Hyperliquid Discord 공식 추천, 프로토콜 협업)
- "레퍼럴 링크 공유" 대신 **콘텐츠 공유** (DeFi Identity 카드, 주간 PnL) 안에 자연스럽게 가입 링크 삽입
- 초기에는 레퍼럴 대신 **프로토콜 퀘스트** 형태로 유저 획득 (Galxe/Layer3 방식)

### F3. "보상 → 등록 → 레퍼럴 루프가 자기 강화적(self-reinforcing)이다"

**PRD 근거:** 전체 비즈니스 플라이휠 (유저 → 데이터 → 광고 → 수익 → 분배 → 유저)

**이것은 F1 + F2의 결합 가설이다:**
```
단계별 전환율이 모두 충분해야 루프가 돈다:

  보상 약속 인지 → 지갑 등록 (F1: 전환율 A)
  등록 후 → 레퍼럴 공유 (F2: 전환율 B)
  공유 → 친구 가입 (전환율 C)
  친구도 → 레퍼럴 공유 (전환율 D)

  루프 성장률 = B × C × D
  예: 30% × 1.5명 × 20% = 0.09 (루프당 0.09명 추가)
  → 이건 바이럴이 아니라 자연 감소
```

**왜 치명적인가:**
- 개별 전환율이 각각 "괜찮아" 보여도, 곱하면 루프가 안 돌 수 있다
- **각 단계의 전환율을 독립적으로 검증하는 것이 아니라, 전체 루프의 감쇠율을 측정해야 한다**

**반론 (틀릴 수 있는 이유):**
- friend.tech의 K-factor가 높았던 이유는 "돈을 벌 수 있다"는 즉각적 투기 동기. HyperView는 3개월 후 $14로 이 수준의 긴급성을 만들 수 없다
- Blur의 시즌 포인트 모델이 성공한 이유는 "에어드롭 토큰의 기대 가치"가 $수천 수준이었기 때문. $14의 기대 가치로는 같은 긴급성 불가
- 자기 강화 루프는 **임계 질량(critical mass)** 이후에만 작동한다. 임계 질량 도달 전까지는 수동으로 밀어야 한다

**검증 방법:**
```
Phase 0-1 전체 퍼널 트래킹:

  유입(랜딩) → 스캔 시작 → 스캔 완료 → 등록
  → 레퍼럴 링크 생성 → 공유 → 유입(레퍼럴)
  → 스캔 → 등록 → 레퍼럴 ...

  각 단계의 전환율을 일별로 추적
  루프 완주율 = (레퍼럴로 가입한 유저 중 다시 레퍼럴하는 비율)

킬 기준:
  - 루프 완주율 < 5%: 자기 강화 루프 불가. 성장은 지속적 외부 유입에 의존
  - 2세대 레퍼럴(내가 초대한 유저가 초대한 유저) 비율 < 1%: 2단계 레퍼럴 무의미
```

**대안 (틀렸을 때):**
- 자기 강화 루프를 포기하고, **콘텐츠 마케팅 + SEO + 파트너십**으로 지속적 외부 유입
- 레퍼럴을 성장 엔진이 아닌 **보조 채널**로 격하
- "바이럴 성장"이 아닌 "유저당 높은 LTV"를 핵심 전략으로 전환

### F4. "프로토콜 광고/어필리에이트로 분배할 만큼의 수익이 발생한다"

**PRD 근거:** 수익 모델 전체 (Section 1.5, 1.6)

**왜 치명적인가:**
- 수익이 없으면 USDC 분배가 없다
- 분배가 없으면 F1(보상 약속)이 거짓이 된다
- 약속이 거짓이면 유저 이탈 → 데이터 감소 → 광고 가치 하락 → 악순환

**구체적 숫자로 검증:**
```
PRD Phase 0-1 시나리오:

  유저 1,000명
  유저당 월 거래 볼륨: ?
  어필리에이트 수수료율: 0.01-0.1%

  낙관: 1,000명 × $20K/월 × 0.05% = $1,000/월
  중립: 1,000명 × $10K/월 × 0.02% = $200/월
  비관: 1,000명 × $5K/월 × 0.01% = $50/월

  유저 풀(70%):
  낙관: $700/월 → 유저당 $0.70/월 = $2.10/분기
  비관: $35/월 → 유저당 $0.035/월 = $0.10/분기

  → PRD의 "$14/분기"를 달성하려면 월 $6,667 수익 필요
  → 1,000명으로 이게 가능한가?
```

**검증 방법:**
```
Phase 0 (제품 전):
  1. 주요 프로토콜 어필리에이트 프로그램 조사
     - Uniswap, Aave, GMX, 1inch 등의 실제 수수료율 확인
     - 최소 전환 요건 확인 (일부는 최소 볼륨/KYC 요구)
  2. 경쟁 플랫폼의 어필리에이트 수익 추정
     - DefiLlama, Zapper 등이 어필리에이트를 쓰는지 조사

Phase 1 (제품 출시 후):
  3. 100명 유저의 실제 어필리에이트 수익 실측
     - 유저당 월 수익 산출
     - 연간 환산 시 의미 있는 분배가 가능한지 판단

킬 기준: 유저당 월 어필리에이트 수익 < $0.50이면
  → 어필리에이트만으로 의미 있는 분배 불가
  → 프로토콜 직접 영업(Featured Listing)을 앞당기거나 수익 모델 전환
```

### F5. "유저가 '아직 증명되지 않은' 신규 플랫폼의 보상 약속을 신뢰한다"

**PRD 근거:** 전체 온보딩 플로우가 "보상 약속"에 의존 (Section 6.1)

**왜 치명적인가:**
- F1(보상이 등록을 유도한다)의 전제 조건이 **신뢰**
- 신뢰 없이는 보상 약속이 등록 동기가 되지 않는다
- 오히려 "USDC 준다"는 메시지가 **스캠 시그널**로 작용할 수 있다

**반론 (틀릴 수 있는 이유):**
- 크립토 유저의 기본 태도는 **불신**이다. "지갑 등록하면 돈 준다"는 피싱과 구분이 안 된다
- Revenue Vault가 온체인에 있어도, Season 1이 끝나기 전까지는 "빈 컨트랙트"다
- "투명한 분배"를 주장하지만, 아직 한 번도 분배한 적이 없다
- 팀이 익명이면 신뢰 불가. 팀이 실명이면 "왜 이런 걸 하지?"라는 의심

**현실 시나리오:**
```
유저 반응 스펙트럼:

  "USDC 준다고? 스캠 아님?" ─── 대다수의 첫 반응
  "어차피 지갑 주소는 퍼블릭이니 손해 볼 건 없지" ─── 합리적 유저
  "얼마나 주는데? $14? 그것 때문에 새 플랫폼 써?" ─── 계산하는 유저
  "한번 해볼까, 어차피 무료니까" ─── 실제 전환되는 유저
```

**검증 방법:**
```
실험 1: 스캠 인식 테스트

  랜딩 페이지를 DeFi Discord/Twitter에 공유
  댓글/반응 관찰:
  - "scam" / "rug" / "too good to be true" 비율
  - 긍정 반응 vs 부정 반응 비율

  킬 기준: 부정 반응 > 50%이면 메시지 전략 수정
  → "USDC 보상"을 전면에서 빼고 유틸리티 우선

실험 2: 신뢰 요소 A/B 테스트

  랜딩 페이지에 아래 신뢰 요소 조합 테스트:
  A: Revenue Vault 컨트랙트 주소 공개
  B: 팀 소개 (실명/경력)
  C: 오디트 리포트
  D: 초기 시드 투자자
  → 어떤 조합이 전환율을 가장 높이는지

실험 3: "무손실 프레이밍"

  "보상을 받는다" 대신 "잃을 것이 없다" 프레이밍:
  "지갑 주소는 이미 퍼블릭입니다. 등록은 무료이고 리스크가 없습니다.
   만약 분배가 실행되면 USDC를 받게 됩니다."
  → 기대를 낮추되 행동 장벽도 낮추는 전략

킬 기준: 어떤 신뢰 요소를 추가해도 전환율 < 10%이면
  → 보상 약속 기반 온보딩 포기
  → 순수 유틸리티(트래커/세금) 기반으로 전환
```

### C1. "HyperEVM 유저가 유효한 비치헤드이고, 멀티체인으로 확장 가능하다"

## 결론: 수정된 핵심 리스크

### 1. 신뢰 콜드스타트 문제 (가장 큰 리스크)

"보상 → 등록 → 레퍼럴" 루프의 시작점은 **신뢰**인데, 신뢰는 **실적으로만 쌓인다.**
Season 1 첫 분배가 실행되기 전까지 모든 것은 약속이다.
크립토 유저에게 "약속"은 가장 싼 화폐이다.

**추천:** Season 1 분배 전까지는 보상을 전면에 내세우지 말고,
**유틸리티(무료 트래커/세금)**로 유저를 모은 뒤, 첫 분배를 서프라이즈로 제공하라.
"약속 → 기대 → 실망" 보다 "무료 도구 → 서프라이즈 보상 → 입소문"이 더 강하다.

### 2. 보상 크기의 딜레마

유저가 적으면 → 광고 수익 적음 → 분배 금액 적음 → 매력 없음 → 유저 안 옴
유저가 많으면 → 분배 금액 나눠짐 → 인당 금액 적음 → 매력 없음

이 딜레마를 풀려면:
- **초기에는 유저 수를 의도적으로 제한**하여 인당 분배 금액을 높이거나
- **보상 외의 가치**(도구, 커뮤니티, 정보)로 리텐션을 해결해야 한다

### 3. Phase 0의 진짜 목표 (수정)

기존: "DeFi Identity 카드 트위터 공유율 > 10%"

**수정: "HyperEVM 활동 분석만으로 지갑 등록 전환율 > 15%"**

이것이 확인되면:
→ "HyperEVM 트랜잭션 해석 + 활동 측정" 자체에 가치가 있다
→ 보상은 성장 가속기로 쓸 수 있다
→ "활동 분석 → 측정 → 보상"이 자연스러운 플라이휠이 된다

이것이 실패하면:
→ HyperEVM 활동 분석만으로는 충분한 등록 동기가 안 된다
→ 보상을 더 전면에 내세우거나, 비치헤드/컨셉 피봇이 필요하다
